Lecture Notes: Data Communications and Networking

COMP3003 Data Communications and Networking

Dr. Ricky Yuen-Tan HOU

Chapter 3: Data Link Layer

1. Overview

数据链路层是 OSI 模型中的第二层，位于物理层之上，主要负责在相邻节点之间实现可靠的数据传输。
数据链路层的三个核心功能是：

  • 帧封装（Framing）：将比特流划分为帧，使得每个帧在传输过程中可以被区分和识别。
  • 错误控制（Error Control）：检测并纠正数据传输中的错误，确保数据完整性。
  • 流量控制（Flow Control）：管理发送和接收端的速度，避免接收端被发送端的数据压垮。

2. Framing

  • Definition: Dividing a bit stream into discrete frames for transmission.
  • Methods:
    1. Character Count:
      • A header field specifies the number of characters in the frame.
      • Pros: Simple.
      • Cons: Vulnerable to errors in the count field.
    2. Flag Bytes with Byte Stuffing:
      • Frames start/end with special flag bytes (e.g., unique patterns).
      • If flag byte appears in data, an escape byte (ESC) is inserted before it.
      • Example: Data “FLAG” → “ESC FLAG”.
    3. Starting/Ending Flags with Bit Stuffing:
      • Uses a special bit pattern (e.g., 01111110) to mark frame boundaries.
      • Inserts a 0 after five consecutive 1s in data to avoid mimicking the flag.
      • Example: Data “011111” → “0111110”.

3. Error Control

错误控制机制的目的是检测和纠正传输中的错误，确保数据的准确性。

Error Detection

  1. Parity Check:
    • Single-Bit Parity:
      • Adds a parity bit to make the total number of 1s even (even parity) or odd (odd parity).
      • Limitation: Detects only single-bit errors.
    • Two-Dimensional Parity:
      • Data arranged in a matrix; parity computed for each row and column.
      • Advantages: Detects and corrects single-bit errors using row/column indices.
  2. Cyclic Redundancy Check (CRC):
    • Treats bit streams as polynomials (e.g., $101 = x^2 + 1$).
    • Uses modulo-2 arithmetic (no carry/borrow).
    • Steps:
      1. Sender and receiver agree on a generator polynomial $G(x)$.
      2. Append $r$ zero bits (where $r =$ degree of $G(x)$) to the frame.
      3. Divide frame by $G(x)$ using modulo-2 division; append remainder as checksum.
      4. Receiver divides received frame by $G(x)$; non-zero remainder = error.
    • Strength: Detects burst errors effectively.

Error Correction

错误纠正是通过冗余位让接收方在发生错误时进行纠正。常用的错误纠正方法包括：

  1. Forward Error Correction (FEC)
  • 原理：发送方在数据中添加冗余位，接收方可以在没有请求重传的情况下，依据冗余位来纠正错误。
  • Maps m-bit data into n-bit codewords ($n = m + r$, where r = redundant bits).
  • Hamming Distance: Number of bit differences between two codewords (e.g., 00000 vs. 00111 = 3).
  • Receiver corrects errors by selecting the nearest valid codeword.
  1. Hamming Code
  • Corrects single-bit errors.
  • Structure:
    • Check bits at positions 1, 2, 4, 8, etc. (powers of 2).
    • Data bits in remaining positions.
  • Computation:
    • Number of check bits ($r$) satisfies: $2^r \ge m + r + 1$ ($m$ = data bits).
    • Each check bit ensures parity for a specific bit combination.
  • Example: For 7 data bits “1001000”:
    • Check bits at 1, 2, 4; compute parity; detect/correct errors via counter.

4. Flow Control

  • Purpose: Prevents sender from overwhelming receiver.
  • Approaches:
    1. Feedback-Based: Receiver sends status updates to sender.
    2. Rate-Based: Built-in mechanism limits transmission rate.
  • Protocols:
    1. Stop-and-Wait Protocol:
      • Sender transmits one frame, waits for acknowledgment (ACK).
      • Part of Automatic Repeat reQuest (ARQ).
      • Disadvantage: Inefficient for high bandwidth/delay networks.
    2. Sliding Window Protocols:
      • One-Bit Sliding Window: Stop-and-Wait with window size 1.
      • Go-Back-N:
        • Window size > 1; sender retransmits all frames after an error.
        • Useful for high bandwidth-delay products.
      • Selective Repeat: Retransmits only erroneous frames (more efficient).

5. 多重访问协议（Multiple-Access Protocols）

多个设备共享同一通信信道时，需要使用多重访问协议来调度信道的使用：

  • Purpose: Manage access to a shared communication channel.
  • Categories:
    1. Channelization:
      • FDMA (Frequency-Division Multiple Access): Divides bandwidth by frequency.
      • TDMA (Time-Division Multiple Access): Divides bandwidth by time slots.
      • CDMA (Code-Division Multiple Access): Uses unique codes for each station.
    2. Random Access (Contention-Based):
      • Aloha: Transmit anytime; retransmit on collision.
      • Slotted Aloha: Transmit only at slot start; reduces collision probability.
      • CSMA (Carrier Sense Multiple Access):
        • Persistent: Sense channel; transmit when idle.
        • Non-Persistent: Wait random time if busy, then retry.
      • CSMA/CD: Detects collisions during transmission; aborts and retries.
    3. Controlled Access (Collision-Free):
      • Polling: Master node controls transmission order.
      • Token-Passing: Token holder transmits; passes token when done.
      • Reservation

6. 扩频技术（Spread Spectrum）

扩频技术用于减少干扰并提高通信安全性，常见的扩频方式有：

6.1 跳频扩频 Frequency-Hopping Spread Spectrum (FHSS)

  • 原理：信号在多个频率之间跳跃，避免干扰和窃听。
    • Hops between frequencies using a spreading code.
    • Receiver syncs with transmitter to recover data.
  • 优点：提高抗干扰能力和安全性。

6.2 直接序列扩频 Direct Sequence Spread Spectrum (DSSS)

  • 原理：通过使用扩频码将信号扩展到更宽的频带，提高抗干扰能力。
    • Spreads signal across a wider band using a spreading code.
    • XORs data with code for transmission; reverses for reception.
  • 优点：减少信号衰减和干扰。

Chapter 4: Network and Transport Layers

Overview

  • Purpose: The Network Layer handles logical addressing and routing of packets across networks, while the Transport Layer ensures end-to-end communication, reliability, and flow control.
  • Key Topics:
    1. IP Addressing (IPv4 and IPv6)
    2. Network Routing
    3. Transport Protocols (TCP and UDP)
    4. Congestion Control
    5. Quality of Service (QoS)

1. IP Addressing

IPv4

  • Structure: 32-bit address, written in dotted decimal (e.g., 158.182.7.1).
  • Header: 20-byte fixed part + optional variable part.
    • Fields: Version (4), IHL (header length), Type of Service (ToS), Total Length, Identification, Flags (DF, MF), Fragment Offset, Time-to-Live (TTL), Protocol, Header Checksum, Source/Destination Addresses.
  • Classful Addressing: Divided into five classes (A-E).
  • Subnetting: Splits networks using subnet masks.
  • Private Addresses: Uses NAT (Network Address Translation) to map private IPs (e.g., 10.0.0.0) to public IPs.

IPv6

  • Structure: 128-bit address, written as eight groups of four hexadecimal digits (e.g., 8000:0000:89AB:CDEF:8000:0000:89AB:CDEF).
  • Header: 40-byte fixed header.
    • Fields: Version (6), Traffic Class, Flow Label, Payload Length, Next Header, Hop Limit, Source/Destination Addresses.
  • Advantages: Larger address space, better QoS, no fragmentation by routers.
  • Transition from IPv4:
    • Dual Stack: Supports both IPv4 and IPv6.
    • Tunneling: Encapsulates IPv6 packets in IPv4.
    • Header Translation: Converts between formats.

2. Network Routing

Delivery

  • Direct Delivery: Packet sent within the same network.
  • Indirect Delivery: Packet routed through multiple networks.
  • Forwarding: Uses routing tables to determine the next hop.

Routing Algorithms

  1. Shortest Path Routing (Dijkstra Algorithm):
    • Finds the shortest path using a graph (nodes = routers, edges = links).
    • Steps: Set initial node to 0, others to infinity; update tentative distances; mark nodes as visited; repeat until done.
  2. Flooding:
    • Sends packets on all outgoing lines except the incoming one.
    • Uses hop counter to limit duplicates.
  3. Distance Vector Routing:
    • Each router maintains a table of distances to destinations, updated via neighbor exchanges.
    • Issue: Count-to-Infinity problem (slow to adapt to bad news).
  4. Link State Routing:
    • Routers flood link state packets (neighbors, costs); each builds a subnet graph to compute routes.
    • Steps: Learn neighbors, measure line costs, build/distribute packets, compute routes.

Special Routing

  • Broadcast: Sends to all destinations (e.g., flooding).
  • Multicast: Sends to a group (via unicast emulation or router destination lists).
  • Hierarchical Routing: Divides routers into regions for scalability.

3. Transport Layer

TCP (Transport Control Protocol)

  • Type: Connection-oriented, reliable.
  • Operation:
    • Sender starts a timer with each segment; retransmits if timeout occurs.
    • Receiver acknowledges the next expected sequence number.
  • Segment Header: Source/Destination Ports, Sequence/Acknowledgment Numbers, Header Length, Flags (e.g., FIN), Window Size, Checksum, Options.
  • Connection:
    • Establishment: Three-way handshake.
    • Release: FIN bit set by either party.
  • Timer Management: Timeout = Mean RTT + Standard Deviation (adjusted dynamically).

UDP (User Datagram Protocol)

  • Type: Connectionless, unreliable.
  • Header: 8 bytes (Source/Destination Ports, Length, Checksum).
  • Use: Simple, low-overhead applications (e.g., DNS, streaming).

4. Congestion Control

Basics

  • Definition: Techniques to prevent or remove network congestion when load exceeds capacity.
  • Categories:
    • Open-Loop: Preventive (e.g., traffic shaping).
    • Closed-Loop: Reactive (e.g., backpressure, choke packets).

Mechanisms

  1. TCP Congestion Control:
    • Window size grows exponentially until a threshold (initially 64 KB), then linearly.
    • On timeout, threshold halves, and window resets to one.
  2. Load Shedding:
    • Routers discard packets when overloaded (randomly or via RED algorithm).
    • RED (Random Early Detection): Drops packets before severe congestion based on queue length.
  3. Metrics: Packet loss, queue lengths, retransmissions, delay, jitter.

Data Traffic

  • Focus: Congestion control avoids traffic jams; QoS ensures suitable conditions.
  • Profiles: Bursty, steady, or variable traffic.

5. Quality of Service (QoS)

Requirements

  • Varies by application:
    • E-mail: High reliability, low delay/bandwidth.
    • Video on Demand: Low reliability, high jitter/bandwidth.
    • Telephony: Low reliability, high delay/jitter.

Techniques

  • Buffering: Reduces jitter (non-real-time only).
  • Traffic Shaping: Smooths traffic flow.
  • Resource Reservation: Allocates bandwidth.
  • Admission Control: Limits new connections.

Jitter Control

  • Definition: Variation in packet arrival times.
  • Solution: Buffering for non-real-time apps (e.g., video-on-demand).

Key Concepts

  • Packet Encapsulation: Layers add headers (e.g., Ethernet, IP, UDP, RTP for multimedia).
  • Network Layer: Logical addressing, routing.
  • Transport Layer: End-to-end delivery, reliability.

Chapter 6: Ethernet and Wireless LANs

1. Standardization of LANs (IEEE Project 802)

  • IEEE Project 802
    • Initiated in 1985 by the IEEE Computer Society to develop interoperable LAN standards covering the Physical and Data Link layers.
    • Defines sub-standards 802.x for different media and technologies (e.g., 802.3 for Ethernet, 802.11 for Wi-Fi).

2. Ethernet (IEEE 802.3)

2.1 History and Evolution

  • Origin (1976) at Xerox PARC; commercialized as “Standard Ethernet” at 10 Mbps.
  • Successive enhancements to higher speeds:
    • Fast Ethernet (802.3u) – 100 Mbps
    • Gigabit Ethernet (802.3z) – 1 Gbps
    • 10/40/100 GbE (802.3ba) – 10–100 Gbps

2.2 Ethernet Frame Format

  • Preamble/SFD: 101010…1011 for bit-level sync.
  • Addresses: 6-byte MAC addresses for source and destination.
  • Type: EtherType field indicates upper-layer protocol (e.g., IPv4=0x0800).
  • Data & Pad: Minimum frame length 64 bytes; pad fills if Data < 46 bytes.
  • FCS: 32-bit CRC for error detection.

2.3 MAC Addressing

  • Notation: E.g., 08-00-20-0A-8C-2B in hexadecimal.
  • Unicast vs. Multicast:
    • LSB of first byte = 0 → unicast; = 1 → multicast.
    • Broadcast: Multicast address with all bits = 1 (FF:FF:FF:FF:FF:FF).

2.4 Physical Encoding

  • Manchester Encoding (10 Mbps): each bit has a mid-bit transition—logical ‘1’ → high→low; ‘0’ → low→high.
  • Fast Ethernet: 4B/5B + MLT-3 (100 Mbps over twisted pair).
  • Gigabit Ethernet: 8B/10B or PAM-5 depending on medium.

2.5 Media-Access Control: CSMA/CD

  1. Carrier Sense: Listen before transmitting.
  2. Multiple Access: All stations share the medium.
  3. Collision Detection (CD): While sending, monitor for other signals; if collision, abort and send 48-bit jam.
  4. Binary Exponential Back-off: After i collisions, wait _k_·slot_time, where k is uniform in [0, 2ᶦ–1], capped at i = 10; give up after 16 tries.

3. Token Ring (IEEE 802.5)

  • Token-Passing: A special 3-byte “token” circulates; possession grants transmission right.
  • Frame Types:
    • Token frames: idle, circulating.
    • Data/Command frames: start-of-frame indicated by toggling one bit.
  • Priority & Reservation:
    • Frames/tokens carry priority levels; only stations with ≥ token priority can seize it.
    • Higher-priority stations can reserve the next token.
  • Fault Management: Active monitor station on ring provides timing, ring-maintenance.

4. Network Devices

Device Layer Function
Hub 1 Physical repeater; regenerates signals to all ports.
Bridge 2 Forwards/filter frames based on MAC; usually 2–4 ports.
Switch 2 Multi-port bridge; MAC learning; full-duplex per port.
Router 3 Forwards packets based on network-layer (IP) addresses.
  • Bridge vs. Router: Routers separate broadcast domains; bridges extend LANs.

5. Wireless LANs (IEEE 802.11)

5.1 Frequency Bands & Channels

  • Operates in the 2.4 GHz ISM band (and 5 GHz for newer amendments).
  • Channels 1–11 in North America; 1–13 in most of world; 1–14 in Japan (14 only for 802.11b).

5.2 Network Architecture

  • Basic Service Set (BSS):
    • Ad hoc (IBSS): Peer-to-peer, no AP.
    • Infrastructure: Stations connect via Access Point (AP).
  • Extended Service Set (ESS): Multiple BSSs interconnected by distribution system (DS) for roaming.

5.3 MAC Layer Challenges

  • Hidden Node: Two stations outside each other’s range both try to send to AP → collision.
  • Exposed Node: Station defers transmission unnecessarily due to hearing unrelated nearby traffic.

5.4 CSMA/CA with RTS/CTS

  1. RTS (Request to Send): Sender → AP/receiver.
  2. CTS (Clear to Send): AP/receiver → Sender (and overheard by others).
  3. Data: Sender transmits frame.
  4. ACK: Receiver acknowledges.
  • Network Allocation Vector (NAV): Virtual carrier-sense timer set by RTS/CTS overhearing to avoid transmission.

5.5 Point Coordination Function (PCF)

  • Polling by AP (Point Coordinator): AP periodically sends beacons and polls stations in contention-free period.
  • Guaranteed transmission slots; no collisions.

6. Bluetooth (IEEE 802.15.1)

  • Ad hoc WPAN in 2.4 GHz band; low-power, short-range (≈10 m).
  • Piconet: One master + up to 7 active slaves (up to 255 parked).
  • Scatternet: Multiple piconets bridged by shared devices.
  • Data rates: 1–24 Mbps (depending on version); built-in encryption.

Key Takeaways

  • IEEE 802 sub-standards ensure LAN interoperability.
  • Ethernet remains dominant, evolving from 10 Mbps to 100 Gbps+.
  • CSMA/CD underpins Ethernet’s access control; CSMA/CA (with RTS/CTS) addresses wireless challenges.
  • Token Ring employs deterministic token-passing, with built-in priority and fault-management.
  • Wireless LANs and Bluetooth extend connectivity untethered, each with tailored MAC protocols and topologies.
